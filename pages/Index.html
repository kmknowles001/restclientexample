    Data Services

    Provide an atomic, context specific and low level access to a specific data domain e.g. Index data. Unlike other market data providers, services can be taken individually or in groups. Unlike other data architecture approaches that try to tackle complex data problems within a single instance "data warehouse", data services break this complexity down to service level. This has many benefits, the primary being that it only has to deliver a single data domain i.e. the solution (service) only looks to solve one problem and therefore can be optimised to do so. This includes improved testing and therefore change, simplier architecture and support etc.
